# Task Decomposition / Task Chunking

## Objetivo
Dividir problemas grandes en unidades de trabajo pequeñas, manejables y verificables.

## Reglas
1. **Atomicidad**: Cada subtarea debe ser completable por sí misma.
2. **Secuencialidad**: No avanzar al siguiente paso sin cerrar el anterior.
3. **Verificabilidad**: Cada paso debe tener un criterio de éxito claro.
4. **Mínima Complejidad**: Buscar la división más simple posible.
